X-Workflow

2019 | Java
Interactive Workflow Management System

The project is designed for supervisor and supervisee to manage work, including identifying work, planning work, scheduling work, executing work, recording work and analyzing performance. Meanwhile, people can post work issues and share work experience in a public sharing space, which means that work issues can be resolved, and work experience can be shared in a timely manner

Overview

System Interface

Data Management

Purpose and Scope

-> Primary Business Driver

The Interactive Workflow Management System project is designed for supervisor and supervisee to manage work, including identifying work, planning work, scheduling work, executing work, recording work and analyzing performance. Meanwhile, people can post work issues and share work experience in a public sharing space, which means that work issues can be resolved, and work experience can be shared in a timely manner.

-> Key Objectives

Work issues will be resolved, and work experience will be shared in a timely manner.

-> In Scope

The public sharing space of the Interactive Workflow Management System is designed within organization-based scope currently

-> Out of Scope

The public sharing space of the Interactive Workflow Management System will be designed within country-based scope in the future

Product Context

User Characteristics

Relative Supervisor: Identifying, Planning, Scheduling, Sharing
Relative Supervisee: Executing, Recording, Analyzing, Sharing

Usability

The user instruction and help guides should be complete
The user instruction should be context sensitive and explain how to achieve common tasks

Performance

-> Static numerical requirements

The number of terminals to be supported no less than 100 for each department in its internal management system
The number of simultaneous users to be supported no less than 100 for each department in its internal management system
Users are classified into different level with different authorizations
Contents about workflow can’t be reviewed by people who is under lower level in its internal management system
Identifying Work Page includes mission purpose, reference cases
Planning Work Page includes mission duration, mission staffing and budget
Scheduling Work Page includes mission completed status and evaluation
Executing Work Page includes mission completed status and uploaded documents
Recording Work Page includes mission completed date and evaluation feedback

-> Dynamic numerical requirements

The number of transactions and the amount of data to be processed within certain time period in external public space to be supported no less than 3000.

Availability

Hours of operation: 23 hours for system operation and 1 hours for system maintenance
Coverage for geographic areas: organization-based scope area (using domain network)
Impact of downtime on users and business operations: data loss and productivity loss

Latency

The maximum acceptable time (or average time) for a service request：30s


Manageability

Monitoring and Maintenance

Information about service health, failure conditions, error detection will be recorded in a specific log in the remote server and the maintenance will be executed later according to the error log

Operations

periods of interactive operations：24hours except 12pm-13pm on Sunday
periods of unattended operations: 12pm-13pm on Sunday
data processing support functions: Data can’t not be change after executing the process of recording. The only way to change the data is to ask for the authorizations from relative supervisor.
backup and recovery operations: System will automatically backup the data
